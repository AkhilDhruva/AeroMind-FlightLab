# assets/

Media for the README and case study.

## What to add

- **`screenshots/`** — capture stills from the running simulator. Good shots:
  - Hero: drone hovering with the HUD visible
  - A lesson mid-flight with the movement/thrust arrows showing
  - The Input → Physics → Explanation feed reacting to a keypress
  - A failure scenario with warnings + Return-to-Home active
  - Anatomy mode with a component tooltip open

- **`preview.gif`** — a short (5–10 s) screen recording of a flight, used as the README hero. Record with the OS screen recorder or [ScreenToGif](https://www.screentogif.com), export as GIF, and drop it here. Then update the badge/image at the top of the root `README.md`.

> `preview.svg` in this folder is a lightweight placeholder poster you can use until a real GIF is recorded.

## Capture tips

- Run the sim full-screen at 1280×720 or larger for crisp stills.
- Use the orbit camera (`C`) for anatomy/hero shots and chase for action.
- Trigger a scenario (Low Battery / Signal Loss) to capture the warning bar and RTH.
